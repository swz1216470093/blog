CPU缓存与Volatile原理

CPU高速缓存

CPU为什么要有高速缓存，是因为最近几十年CPU频率不断提升，而内存在访问速度上却没有质的突破，导致CPU的处理速度和内存的访问速度差矩越来越大。这种情况下传统的 CPU 直连内存的方式显然就会因为内存访问的等待，导致计算资源大量闲置，降低 CPU 整体吞吐量。同时又由于局部性原理，在 CPU 和内存之间加入一层高速缓存就显得性价比极高了。

线程池探究（二）

线程池的状态和状态转换

线程池状态

Running：接收新任务并处理阻塞队列中的任务

Shutdown：拒绝新任务但可以处理阻塞队列中的任务

Stop:拒绝新任务并抛弃阻塞队列中的任务，同时中断正在处理的任务

Tidying:所有任务执行完成（包括阻塞队列中的任务）后，活动线程数为0，将要调用terminated方法

Terminated：终止状态，调用完terminated方法后的状态
